Page of a book

Some users prefer to read a book page by page. Create the handler to serve a page by the request.

You can get n_page variable from the **kwargs argument with the number of the page to show: "1", "2", "3"... Store the page content to the context by the key "content".

Use a predefined book dictionary with pages you will show to the user and render the template defined in file "book/templates/book/page.html".

By default, Django is looking for templates in all of your " <application_name>/templates" directories, so we do not include " book/templates" in the template path, because Django will do it for us.
